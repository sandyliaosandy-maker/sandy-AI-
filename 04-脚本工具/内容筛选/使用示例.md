# 使用示例

## 示例 1: 同步最近30天的内容

### 配置文件 (config/filter-config.json)

```json
{
  "obsidianPath": "/Users/username/Documents/ObsidianVault",
  "tableFile": "内容索引.md",
  "projectContentPath": "../01-源代码/网站代码/内容",
  "filters": {
    "date": {
      "custom": "最近30天"
    }
  }
}
```

### 执行

```bash
npm start
```

## Obsidian 表格示例

在 Obsidian 中创建一个名为 `内容索引.md` 的文件：

```markdown
# 内容索引

| 文件路径 | 日期 | 标题 | 标签 | 评分 | 摘要 |
|---------|------|------|------|------|------|
| 公开内容/新闻/DeepSeek商业化分析.md | 2025-01-06 | DeepSeek 商业化分析 | AI,商业 | 9.5 | 分析 DeepSeek 的商业化路径 |
| 公开内容/笔记/小红书笔记1.md | 2025-01-05 | 我的小红书笔记 | 生活,分享 | 8.0 | 分享生活点滴 |
| 公开内容/新闻/AI行业动态.md | 2025-01-04 | AI 行业动态 | AI | 7.5 | 最新 AI 行业动态 |
```

## 执行流程

1. **准备表格**: 在 Obsidian 中创建或更新 `内容索引.md` 文件
2. **配置路径**: 编辑 `config/filter-config.json`，设置正确的路径
3. **设置筛选规则**: 根据需要配置筛选条件
4. **执行同步**: 运行 `npm start` 或 `npm run sync:content`
5. **查看结果**: 检查项目 `内容/` 目录，确认文件已同步

## 常见问题

### Q: 如何找到 Obsidian 知识库的路径？

A: 在 Obsidian 中，点击设置 → 文件与链接 → 查看"库文件夹位置"

### Q: 表格中的文件路径怎么写？

A: 可以使用相对路径（相对于 Obsidian 知识库根目录），例如：
- `公开内容/新闻/文章.md`
- `公开内容/笔记/笔记.md`

也可以使用绝对路径（不推荐）

### Q: 日期格式有哪些？

A: 支持以下格式：
- `YYYY-MM-DD` (推荐)
- `YYYY/MM/DD`
- `YYYY.MM.DD`

### Q: 如何禁用增量同步？

A: 在配置文件中设置：
```json
{
  "syncOptions": {
    "incrementalSync": false
  }
}
```
