# 内容审查流程

## 背景

当前 Obsidian 知识库只有 md 文件，没有人工审查编辑能力。本文档定义如何建立内容审查流程。

## 审查流程设计

### 流程概览

```
Python 萃取系统
    ↓
生成 Markdown 文件
    ↓
Obsidian /收件箱/        ← 草稿状态
    ↓
人工审查
    ↓
审查通过？
    ├─ 是 → 移动到 /公开内容/新闻/ 或 /公开内容/笔记/
    └─ 否 → 删除或修改后重新审查
    ↓
Obsidian Git 插件自动提交
    ↓
GitHub 仓库
    ↓
Vercel 自动构建
    ↓
网站更新
```

## 审查标准

### 1. 内容质量标准

#### 必需项
- [ ] 内容完整（有标题、正文）
- [ ] 标题清晰明确
- [ ] 内容有价值（不是垃圾内容）
- [ ] 没有明显的错误或错别字

#### 推荐项
- [ ] 有摘要（summary）
- [ ] 有标签（tags）
- [ ] 有评分（score，如适用）
- [ ] 格式正确（Markdown 格式）

### 2. Frontmatter 标准

#### News 文章 Frontmatter
```yaml
---
title: "文章标题"          # 必需
date: 2025-01-06          # 必需
tags: [AI, 商业]          # 推荐
score: 9.5                # 可选
summary: "一句话摘要"      # 推荐
---
```

#### Note 文章 Frontmatter
```yaml
---
title: "笔记标题"          # 必需
date: 2025-01-06          # 必需
tags: [生活, 分享]         # 推荐
source: "xiaohongshu"     # 可选
---
```

### 3. 内容格式标准

- [ ] 使用标准 Markdown 语法
- [ ] 图片路径正确（使用 `/attachments/` 绝对路径）
- [ ] 没有 Obsidian 特定语法（如 `[[链接]]`）
- [ ] 代码块有语言标识
- [ ] 链接格式正确

## 审查操作步骤

### 步骤 1: 查看收件箱内容

1. 打开 Obsidian
2. 进入 `收件箱/` 目录
3. 查看新生成的内容文件

### 步骤 2: 审查内容

1. **快速浏览**：判断内容是否有价值
2. **详细审查**：
   - 检查内容完整性
   - 检查标题是否清晰
   - 检查是否有错误
   - 检查格式是否正确
3. **补充信息**：
   - 添加或修改 Frontmatter
   - 添加标签
   - 添加摘要
   - 添加评分（如适用）

### 步骤 3: 处理审查结果

#### 审查通过
1. 在 Obsidian 中，将文件移动到对应目录：
   - 新闻内容 → `公开内容/新闻/`
   - 笔记内容 → `公开内容/笔记/`
2. 确保 Frontmatter 完整
3. 确保图片在 `公开内容/附件/` 目录

#### 审查不通过
1. **删除**：如果内容没有价值，直接删除
2. **修改**：如果有价值但需要修改，修改后重新审查
3. **暂存**：如果不确定，可以暂时保留在收件箱

### 步骤 4: 自动同步

1. Obsidian Git 插件会自动检测文件变更
2. 自动提交并推送到 GitHub
3. GitHub Webhook 触发 Vercel 构建
4. 网站自动更新

## 审查工具和技巧

### Obsidian 插件推荐

1. **Templater** - 创建 Frontmatter 模板
2. **Tag Wrangler** - 管理标签
3. **Obsidian Git** - 自动 Git 同步

### 审查效率技巧

1. **批量审查**：一次审查多个文件
2. **使用模板**：创建 Frontmatter 模板，快速添加元数据
3. **标签管理**：建立标签体系，快速分类
4. **快捷键**：使用 Obsidian 快捷键提高效率

## 审查频率建议

### 每日审查
- 建议每天审查一次收件箱
- 每次审查 10-20 篇文章
- 保持收件箱内容不超过 50 篇

### 批量审查
- 如果内容较多，可以批量审查
- 每周至少审查一次
- 避免收件箱积压过多内容

## 问题处理

### 问题 1: 内容格式不统一
**解决方案**: 建立格式标准，使用模板

### 问题 2: 审查效率低
**解决方案**: 使用模板、快捷键，批量处理

### 问题 3: 收件箱积压
**解决方案**: 定期审查，建立审查计划

### 问题 4: 审查标准不清晰
**解决方案**: 完善审查标准文档，定期更新
