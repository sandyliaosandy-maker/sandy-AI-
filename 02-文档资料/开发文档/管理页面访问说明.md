# 管理页面访问说明

## 访问地址

### 方法 1: 直接访问（推荐）

在浏览器地址栏输入：
```
http://localhost:3002/管理
```

### 方法 2: URL 编码访问

如果直接访问不工作，可以使用 URL 编码：
```
http://localhost:3002/%E7%AE%A1%E7%90%86
```

### 方法 3: 从首页导航

如果导航栏添加了管理链接，可以直接点击。

## 解决端口占用问题

如果遇到 `EADDRINUSE` 错误，需要先关闭占用端口的进程：

```bash
# 1. 查找占用端口的进程
lsof -ti:3002

# 2. 关闭进程（替换 <PID> 为实际的进程ID）
kill -9 <PID>

# 3. 重新启动开发服务器
cd "/Users/luyu/CascadeProjects/Sandy的AI收藏夹/01-源代码/网站代码"
npm run dev
```

## 验证服务器是否正常运行

启动开发服务器后，应该看到类似输出：
```
  ▲ Next.js 14.2.0
  - Local:        http://localhost:3002
  - Ready in X.Xs
```

如果看到这个输出，说明服务器正常运行，可以访问管理页面。

## 常见问题

### 问题 1: 404 错误

**可能原因**:
- 服务器没有正常启动
- 端口被占用
- 路径输入错误

**解决方案**:
1. 确认服务器正在运行（查看终端输出）
2. 检查端口是否被占用
3. 尝试使用 URL 编码的路径

### 问题 2: 端口被占用

**解决方案**: 参考 [端口占用解决方案](./端口占用解决方案.md)

### 问题 3: 页面显示空白

**可能原因**:
- JavaScript 错误
- 组件加载失败

**解决方案**:
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签页的错误信息
3. 检查 Network 标签页，确认资源加载是否成功
