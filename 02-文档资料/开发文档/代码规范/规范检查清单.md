# 规范检查清单

本文档提供开发过程中的规范检查清单，确保代码符合项目规范。

## 提交前检查清单

### 代码质量检查
- [ ] 运行 `npm run type-check` 通过（无类型错误）
- [ ] 运行 `npm run lint` 通过（无 Lint 错误）
- [ ] 运行 `npm run format:check` 通过（格式正确）
- [ ] 运行 `npm run build` 成功（构建通过）

### 代码规范检查
- [ ] 所有组件都有类型定义
- [ ] 所有 Props 都有类型定义
- [ ] 没有使用 `any` 类型（除非必要）
- [ ] 组件命名符合规范（PascalCase）
- [ ] 变量命名符合规范（camelCase）
- [ ] 函数命名符合规范（动词开头）

### 代码风格检查
- [ ] 代码格式统一（Prettier）
- [ ] 没有未使用的导入
- [ ] 没有未使用的变量
- [ ] 没有 console.log（生产代码）
- [ ] 注释清晰（如需要）

### Git 提交检查
- [ ] 提交信息符合约定式提交格式
- [ ] 提交信息清晰描述更改内容
- [ ] 没有提交敏感信息（API 密钥、密码等）
- [ ] 没有提交临时文件和备份
- [ ] 没有提交 `node_modules` 和构建产物

## 功能开发检查清单

### 组件开发
- [ ] 组件符合单一职责原则
- [ ] 组件可复用（如需要）
- [ ] Props 设计合理
- [ ] 有默认值（可选属性）
- [ ] 有错误处理（如需要）
- [ ] 有加载状态（如需要）

### 页面开发
- [ ] 页面有 metadata 定义
- [ ] 页面有错误处理（error.tsx）
- [ ] 页面有 404 处理（not-found.tsx）
- [ ] 页面性能优化（图片优化、代码分割）

### 内容管理
- [ ] Markdown 文件有正确的 Frontmatter
- [ ] 图片路径正确（使用绝对路径）
- [ ] 内容格式正确（标准 Markdown）
- [ ] 没有 Obsidian 特定语法

### 样式开发
- [ ] 使用 Tailwind CSS（不使用内联样式）
- [ ] 使用主题颜色（不硬编码颜色）
- [ ] 响应式设计（移动端优先）
- [ ] 使用 Typography 插件（文章内容）

- [ ] 组件可复用

### 错误处理
- [ ] 有完善的错误处理

- [ ] 有日志记录（开发环境）

## 快速检查命令

```bash
# 完整检查
npm run check

# 单独检查
npm run type-check    # TypeScript 类型检查
npm run lint          # ESLint 检查
npm run format:check  # Prettier 格式检查
npm run build         # 构建检查
```

## 使用建议

1. **提交前**: 使用"提交前检查清单"
2. **开发中**: 使用"功能开发检查清单"
3. **代码审查**: 使用"代码审查检查清单"
4. **功能完成**: 使用"功能完成检查清单"
5. **定期审查**: 使用"定期审查检查清单"
