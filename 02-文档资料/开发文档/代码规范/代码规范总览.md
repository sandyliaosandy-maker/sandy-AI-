# 代码规范总览

本文档为 **Sandy的AI收藏夹** 项目的代码规范总览，提供快速导航到各个具体规范文档。

## 项目概述

**Sandy的AI收藏夹** 是一个基于 Next.js + Obsidian + Contentlayer 的个人知识库网站项目。项目采用静态站点生成（SSG），通过 Git-based Headless CMS 模式管理内容。

### 技术环境
- **框架**: Next.js 14+ (App Router)
- **语言**: TypeScript
- **UI库**: React 18+
- **样式**: Tailwind CSS
- **内容管理**: Contentlayer + Markdown
- **知识库**: Obsidian
- **部署**: Vercel

## 快速参考

### 目录结构规范
```
01-源代码/
├── 网站代码/              # Next.js 项目
│   ├── app/              # App Router（框架要求）
│   ├── components/       # React 组件
│   ├── lib/              # 工具函数
│   └── public/           # 静态资源（框架要求）
└── 内容源/                # Obsidian 内容
    └── 公开内容/
        ├── 新闻/
        ├── 笔记/
        ├── 页面/
        └── 附件/
```

### 核心原则
1. **类型安全**: 充分利用 TypeScript 类型系统
2. **组件化**: 可复用、可维护的组件设计
3. **性能优先**: 静态生成、代码分割、图片优化
4. **内容驱动**: 基于 Markdown 的内容管理
5. **一致性**: 统一的代码风格和命名规范

## 开发流程

1. **需求分析** → 明确功能边界和输入输出
2. **设计接口** → 定义组件接口和类型
3. **实现组件** → 开发 React 组件
4. **内容管理** → 在 Obsidian 中编写内容
5. **测试验证** → 本地测试和构建验证
6. **提交代码** → 遵循 Git 工作流规范
7. **自动部署** → Vercel 自动构建和部署

## 注意事项

- 所有代码必须通过 TypeScript 类型检查
- 组件必须遵循 React 最佳实践
- 样式使用 Tailwind CSS，避免内联样式
- 内容通过 Contentlayer 管理，不直接操作文件系统
- Git 提交遵循约定式提交规范
