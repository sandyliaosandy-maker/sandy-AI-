## 快速参考

### 核心原则
1. **类型安全**: 充分利用 TypeScript 类型系统
2. **组件化**: 可复用、可维护的组件设计
3. **性能优先**: 静态生成、代码分割、图片优化
4. **内容驱动**: 基于 Markdown 的内容管理
5. **一致性**: 统一的代码风格和命名规范

### 命名规范速查
- **组件**: PascalCase (`PostCard.tsx`)
- **变量/函数**: camelCase (`formatDate`)
- **常量**: UPPER_SNAKE_CASE (`MAX_ITEMS`)
- **类型/接口**: PascalCase (`PostCardProps`)
- **文件**: 与组件名一致或 kebab-case

### Git 提交格式
```
<type>(<scope>): <subject>

feat(首页): 添加文章列表组件
fix(笔记页): 修复图片路径问题
docs(README): 更新项目说明
```

## 使用建议

1. **开发前**: 阅读相关规范文档，了解最佳实践
2. **开发中**: 参考规范文档，确保代码符合规范
3. **代码审查**: 使用规范文档作为审查标准
4. **持续更新**: 根据项目发展更新规范文档

## 配置文件位置

实际的代码规范配置文件应放在 `03-配置文件/` 目录：
- `.eslintrc` - ESLint 配置
- `.prettierrc` - Prettier 配置
- `tsconfig.json` - TypeScript 配置
- `tailwind.config.js` - Tailwind CSS 配置
