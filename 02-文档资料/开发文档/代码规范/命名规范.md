# 命名规范

本文档定义项目中的文件、变量、函数等命名规范。

## 文件命名规范

### 1. 组件文件

```typescript
// ✅ 正确 - PascalCase
PostCard.tsx
Button.tsx
Header.tsx

// ✅ 正确 - kebab-case（也可以）
post-card.tsx
button.tsx
header.tsx

// ❌ 错误
postCard.tsx      // 不一致
post_card.tsx     // 不使用下划线
```

### 2. 工具函数文件

```typescript
// ✅ 正确 - camelCase
utils.ts
dateFormat.ts
contentHelper.ts

// ✅ 正确 - kebab-case（也可以）
date-format.ts
content-helper.ts
```

### 3. 配置文件

```typescript
// ✅ 正确 - kebab-case
next.config.js
tailwind.config.js
tsconfig.json
```

### 4. 页面文件

```typescript
// ✅ 正确 - Next.js 约定
app/page.tsx           // 首页
app/layout.tsx         // 布局
app/笔记/page.tsx      // 笔记列表页
app/笔记/[别名]/page.tsx  // 笔记详情页
```

## 变量命名规范

### 1. 变量和函数

```typescript
// ✅ 正确 - camelCase
const userName = 'John'
const postList = []
function formatDate() {}
function getUserData() {}

// ❌ 错误
const user_name = 'John'    // 不使用下划线
const PostList = []         // 不使用 PascalCase（变量）
```

### 2. 常量

```typescript
// ✅ 正确 - UPPER_SNAKE_CASE
const MAX_ITEMS = 10
const API_BASE_URL = 'https://api.example.com'
const DEFAULT_CONFIG = {}

// ✅ 正确 - 对象常量使用 camelCase
const defaultConfig = {
  maxItems: 10,
  apiUrl: 'https://api.example.com',
}
```

### 3. 布尔值

```typescript
// ✅ 正确 - is/has/can/should 前缀
const isLoading = true
const hasError = false
const canEdit = true
const shouldShow = false

// ❌ 错误
const loading = true      // 不够明确
const error = false       // 不够明确
```

## 组件命名规范

### 1. 组件名

```typescript
// ✅ 正确 - PascalCase
export function PostCard() {}
export function NewsList() {}
export function HeroBanner() {}

// ❌ 错误
export function postCard() {}    // 小写
export function Post() {}        // 不够描述性
```

### 2. Props 接口

```typescript
// ✅ 正确 - 组件名 + Props
interface PostCardProps {
  post: News
}

interface ButtonProps {
  children: React.ReactNode
}

// ❌ 错误
interface Props {}              // 不够明确
interface PostCard {}          // 不是 Props
```

## 类型命名规范

### 1. 接口和类型

```typescript
// ✅ 正确 - PascalCase
interface UserData {
  name: string
  email: string
}

type ButtonVariant = 'primary' | 'secondary'

// ✅ 正确 - 泛型使用单个大写字母
interface ListProps<T> {
  items: T[]
}
```

### 2. 枚举

```typescript
// ✅ 正确 - PascalCase，值使用 UPPER_SNAKE_CASE
enum Status {
  PENDING = 'PENDING',
  COMPLETED = 'COMPLETED',
  FAILED = 'FAILED',
}

// ✅ 正确 - 常量对象（推荐）

  PENDING: 'PENDING',
  COMPLETED: 'COMPLETED',
  FAILED: 'FAILED',
} as const
```

## 开发注意事项

### ✅ 正确做法

1. **一致性**：同类命名保持一致性
2. **描述性**：名称清晰表达用途
3. **遵循约定**：遵循语言和框架约定
4. **避免缩写**：除非是通用缩写（如 API、URL）

### ❌ 避免做法

1. **不要**使用拼音命名
2. **不要**使用单字母变量名（除了循环变量）
3. **不要**使用下划线命名（TypeScript/JavaScript 约定）
4. **不要**使用保留字作为名称
5. **不要**使用模糊的名称（如 `data`、`item`、`temp`）
