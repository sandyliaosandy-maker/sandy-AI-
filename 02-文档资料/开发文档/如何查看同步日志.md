# 如何查看同步日志

## 1. 查看浏览器控制台日志

### 步骤：
1. **打开浏览器开发者工具**
   - 按 `F12` 键
   - 或按 `Cmd + Option + I` (Mac) / `Ctrl + Shift + I` (Windows)
   - 或右键点击页面 → 选择"检查" / "Inspect"

2. **切换到 Console（控制台）标签页**
   - 在开发者工具顶部，点击 "Console" 标签

3. **查看日志**
   - 点击同步按钮后，应该能看到以下日志：
     - `按钮被点击`
     - `handleSync 被调用`
     - `选中的条目数量: X`
     - `发送同步请求...`
     - `收到响应: 200 OK`
     - `解析结果: {success: true, stats: {...}}`
     - `同步统计: {totalFiles: X, syncedFiles: X, skippedFiles: X, errors: [...]}`
     - `同步成功！统计信息: {...}`

4. **查找同步统计**
   - 在控制台中搜索 `同步统计` 或 `stats`
   - 或者查看 `解析结果:` 后面的对象，里面有 `stats` 字段

---

## 2. 查看服务器终端日志

### 步骤：
1. **找到运行开发服务器的终端窗口**
   - 这是你运行 `npm run dev` 的终端窗口
   - 通常在 VS Code、终端应用或其他命令行工具中

2. **查看终端输出**
   - 点击同步按钮后，终端应该显示：
     ```
     [API] 收到同步请求
     [API] 请求数据: { selectedRowsCount: X, ... }
     [API] 开始同步文件...
     [同步] 处理文件: 公开内容/新闻/xxx.md
     [同步] 源路径: /Users/luyu/.../xxx.md
     [同步] 文件存在，继续处理
     [同步] 目标路径: /Users/luyu/.../内容/公开内容/新闻/xxx.md
     [同步] 文件复制成功: 公开内容/新闻/xxx.md
     [API] 同步完成: { totalFiles: X, syncedFiles: X, skippedFiles: X, errors: [...] }
     ```

3. **如果文件不存在**
   - 会显示：`[同步] 文件不存在，跳过: /path/to/file.md`
   - 说明源文件路径不正确

---

## 3. 查看页面上的同步结果

### 步骤：
1. **点击同步按钮后，向下滚动页面**
   - 同步结果会显示在页面底部

2. **查看同步结果区域**
   - **成功时**：显示绿色背景的卡片
     - 标题：`同步完成！`
     - 内容：
       - 总计: X 个文件
       - 成功: X 个文件
       - 跳过: X 个文件
       - 错误信息（如果有）

   - **失败时**：显示红色背景的卡片
     - 标题：`同步失败`
     - 内容：错误信息

3. **如果看不到同步结果**
   - 检查页面是否完全加载
   - 尝试刷新页面
   - 检查是否有 JavaScript 错误

---

## 4. 常见问题

### Q: 控制台没有日志？
**A**: 
- 确保开发者工具已打开
- 确保在 Console 标签页
- 尝试刷新页面后重新操作
- 检查是否有 JavaScript 错误阻止了代码执行

### Q: 服务器终端没有日志？
**A**:
- 确保开发服务器正在运行（`npm run dev`）
- 检查终端窗口是否可见
- 尝试重新启动开发服务器

### Q: 页面上没有显示同步结果？
**A**:
- 检查页面是否完全加载
- 查看控制台是否有错误
- 尝试刷新页面
- 检查同步结果区域是否在页面底部（可能需要滚动）
