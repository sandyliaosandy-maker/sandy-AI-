# 端口占用解决方案

## 问题

启动开发服务器时遇到 `EADDRINUSE: address already in use :::3002` 错误，表示端口 3002 已被占用。

## 解决方案

### 方法 1: 关闭占用端口的进程（推荐）

1. **查找占用端口的进程**：
   ```bash
   lsof -ti:3002
   ```

2. **关闭进程**：
   ```bash
   kill -9 <PID>
   ```
   如果有多个进程，分别关闭：
   ```bash
   kill -9 779 41908
   ```

3. **重新启动开发服务器**：
   ```bash
   npm run dev
   ```

### 方法 2: 更换端口

如果无法关闭占用端口的进程，可以更换端口：

1. **修改 package.json**：
   ```json
   {
     "scripts": {
       "dev": "next dev -p 3003",
       "start": "next start -p 3003"
     }
   }
   ```

2. **重新启动**：
   ```bash
   npm run dev
   ```

3. **访问新端口**：
   ```
   http://localhost:3003/管理
   ```

### 方法 3: 使用系统工具（macOS）

```bash
# 查找并关闭占用端口的进程
sudo lsof -ti:3002 | xargs kill -9
```

## 预防措施

开发完成后记得关闭开发服务器（Ctrl+C），避免端口被占用。
