## 遇到的问题

### 问题1: Vercel 部署失败 - 缺失依赖包
- **现象**: 
  - 部署时出现 "Module not found" 错误
  - 缺少 stripe, mysql2, @vercel/postgres, pg, bcryptjs 等依赖

- **根本原因**: 
  - package.json 中缺少支付和数据库相关的依赖包
  - 这些依赖在代码中被引用，但未在 package.json 中声明

- **解决方案**:
  1. 添加缺失的依赖包到 package.json
  2. 添加对应的类型定义包
  3. 重新部署验证

### 问题2: ESLint 代码规范警告
- **现象**: 
  - 未使用的变量警告（createPortalSession, getDatabase, request）
  - any 类型警告（lib/db/adapters/postgres.ts）

- **解决方案**:
  1. 移除未使用的导入
  2. 将未使用的参数重命名为 _request，并添加 ESLint 禁用注释
  3. 为 intentionally 使用 any 的地方添加 ESLint 禁用注释

### 问题3: TypeScript 类型错误
- **现象**: 
  - `Property 'id' does not exist on type 'never'` 在 app/api/subscriptions/check/route.ts
  - getCurrentUser() 返回类型不正确

- **根本原因**: 
  - getCurrentUser() 返回类型被定义为 `Promise<null>`，导致类型推断失败
  - 需要使用正确的联合类型 `Promise<User | null>`

- **解决方案**:
  1. 更新 getCurrentUser() 返回类型为 `Promise<User | null>`
  2. 更新 getUserSubscription() 返回类型为 `Promise<Subscription | null>`
  3. 使用可选链操作符安全访问 currentUser?.id

### 问题4: Stripe API 版本类型错误
- **现象**: 
  - TypeScript 报错：`Type '"2024-11-20.acacia" is not assignable to type "2023-10-16"'`

- **根本原因**: 
  - 安装的 Stripe 库版本不支持 '2024-11-20.acacia' API 版本
  - 需要使用库支持的 API 版本

- **解决方案**:
  - 将 API 版本从 '2024-11-20.acacia' 调整为 '2023-10-16'

### 问题5: /notes 页面 404 错误
- **现象**: 
  - 访问 http://localhost:3003/notes 返回 404

- **解决方案**:
  - 添加 metadata 导出到 app/notes/page.tsx
  - 重启开发服务器，清除 Next.js 缓存

### 问题6: 图片同步失败
- **现象**: 
  - "同步到线上"功能成功同步了文字内容，但图片文件未同步
  - API 返回 "没有需要同步的更改"，但实际有新的图片文件

- **根本原因**: 
  1. Git status 命令在指定目录路径时可能无法正确检测文件
  2. Git 返回的路径包含 Unicode 转义序列（如 `\346\272\220\344\273\243\347\240\201`）
  3. 文件过滤逻辑只检查了周报目录，未包含图片上传目录

- **解决方案**:
  1. **改进 Git 状态检测**:
     - 改为检查整个项目的 Git 状态，而不是只检查特定目录
     - 然后过滤出周报目录和图片上传目录的文件
  2. **增强路径解析**:
     - 支持解析带引号的路径
     - 支持解析 Unicode 转义序列路径
     - 同时检查普通路径和转义序列路径
  3. **优化文件过滤**:
     - 同时检查周报目录（包含"周报"或 Unicode 转义序列）
     - 同时检查图片上传目录（包含"uploads"）
  4. **添加详细日志**:
     - 记录 Git 命令输出
     - 记录过滤后的文件列表
     - 便于调试和排查问题

## 技术细节

### Git 状态检测优化

**改进前的问题**:
```typescript
// 只检查特定目录，可能无法检测到文件
git status --porcelain --untracked-files=all "01-源代码/网站代码/内容/公开内容/周报"
```

**改进后的方案**:
```typescript
// 检查整个项目，然后过滤
git status --porcelain --untracked-files=all
// 然后过滤出相关文件
const filteredLines = allLines.filter((line) => {
  const filePath = line.substring(3).trim().replace(/^["']|["']$/g, '')
  const isNewsletter = 
    filePath.includes('周报') ||
    filePath.includes('\\345\\221\\250\\346\\212\\245') ||
    filePath.includes(NEWSLETTER_DIR)
  const isUploads = 
    filePath.includes('uploads') ||
    filePath.includes(UPLOADS_DIR)
  return isNewsletter || isUploads
})
```

### Unicode 转义序列处理

Git 在处理包含中文字符的路径时，可能返回 Unicode 转义序列（八进制格式）：
- "源代码" → `\346\272\220\344\273\243\347\240\201`
- "周报" → `\345\221\250\346\212\245`

代码中同时检查普通路径和转义序列路径，确保能够正确识别文件。

### 文件同步流程

1. **检测阶段**:
   - 使用 `git status --porcelain --untracked-files=all` 检查整个项目
   - 过滤出周报目录和图片上传目录的文件
   - 如果 git status 没有输出，使用 `git ls-files --others --exclude-standard` 检查未跟踪文件

2. **解析阶段**:
   - 解析 Git 状态输出，提取文件路径
   - 处理带引号的路径和 Unicode 转义序列
   - 分类为新增、修改、删除、未跟踪

3. **提交阶段**:
   - 使用 `git add` 添加周报目录和图片上传目录的文件
   - 提交更改
   - 推送到远程仓库

## 学习笔记

1. **Git 路径处理**:
   - Git 在处理包含特殊字符（如中文）的路径时，可能返回 Unicode 转义序列
   - 需要使用多种方式检查路径，确保能够正确识别文件
   - `git status` 在指定目录路径时可能无法正确检测文件，建议检查整个项目后过滤

2. **TypeScript 类型推断**:
   - 函数的返回类型会影响调用方的类型推断
   - 使用联合类型（如 `User | null`）比单一类型（如 `null`）更准确
   - 可选链操作符（`?.`）可以安全地访问可能为 null 的属性

3. **ESLint 配置**:
   - 对于 intentionally 未使用的参数，可以使用 `_` 前缀
   - 对于 intentionally 使用 `any` 的地方，可以添加 ESLint 禁用注释
   - 保持代码规范的同时，允许必要的例外情况

4. **Stripe API 版本**:
   - Stripe 库的 TypeScript 类型定义限制了可用的 API 版本
   - 需要使用库支持的 API 版本，而不是最新的 API 版本
   - 可以通过查看库的类型定义来确定支持的版本

5. **Next.js 路由**:
   - 页面组件需要导出 `metadata` 对象以支持 SEO 和浏览器标签
   - 某些情况下需要重启开发服务器才能看到路由变更
