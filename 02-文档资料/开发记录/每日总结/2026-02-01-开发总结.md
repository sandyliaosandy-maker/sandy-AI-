## 遇到的问题

### 问题 1：部署时报 `Cannot read properties of undefined (reading 'inTable')`
- **现象**：Creating an optimized production build 阶段大量相同 TypeError，栈指向 mdast-util-from-markdown、remark-parse、@mdx-js/mdx。
- **原因**：remark-gfm v4.0.0 已知问题（GitHub #57），解析含特定表格/反引号内容时访问未定义的 inTable。
- **解决**：在 package.json 将 `remark-gfm` 从 `^4.0.1` 改为 `3.0.1`，执行 `npm install --legacy-peer-deps`。

### 问题 2：MDX 报 `Unexpected character '1'/'2' before name`
- **现象**：两篇新闻（海信 E8S、炎和科技）编译 MDX 时报“期望字母/`$`/`_` 作为名称开头，遇到数字”。
- **原因**：正文中 `<1`、`<20` 等被 MDX 解析为 JSX 标签开头，标签名不能以数字开头。
- **解决**：在对应 .md 中将 `<1`→`&lt;1`、`<20`→`&lt;20`（frontmatter 与正文均需一致）。

### 问题 3：`Expected a closing tag for <br>`
- **现象**：聚焦Z世代一文报“paragraph 结束前应有 `<br>` 的闭合标签”。
- **原因**：MDX/JSX 要求标签闭合，`<br>` 需写成自闭合形式。
- **解决**：该文件内所有 `<br>` 替换为 `<br />`。

### 问题 4：Contentlayer 报 document 字段与定义不匹配
- **现象**：聚焦Z世代一文报 date 字段 incompatible（值为标题字符串）。
- **原因**：frontmatter 中 date、source 填错列（date 填成标题，source 填成日期）。
- **解决**：修正该文件 frontmatter：`date: 2026-01-07`，`source: "36氪"`。

### 问题 5：remarkGfm 与 react-markdown 类型不兼容
- **现象**：TypeScript 报 remark 插件类型不能赋值给 Pluggable，涉及两套 vfile/vfile-message 类型（react-markdown 依赖与项目依赖）。
- **解决**：传插件时使用断言 `remarkPlugins={[remarkGfm as any]}`，仅绕过类型检查，运行时行为不变。

## 学习笔记
- **remark-gfm 版本**：v4 存在 inTable 未定义 bug 且官方未修，内容站若大量表格/反引号，可优先用 3.x。
- **MDX 与正文**：内容里的 `<` 会按 JSX 解析，`<数字`、`<字母` 都可能被当标签，数字开头标签非法；需用 `&lt;` 或代码块包裹。
- **JSX 标签**：在 MDX 中使用的 HTML 标签需符合 JSX 规范，如 `<br>` 需写成 `<br />`。
- **类型与插件**：多依赖共用 unified/vfile 时易出现类型不兼容，对插件做 `as any` 是常见且安全的折中。
