# 开发记录目录

本目录存放项目开发过程中的任务总结、进度记录和 Checkpoint 记录。

## 目录结构

### 每日总结/
存放每日开发任务总结，按日期组织。

**文件命名格式**：`YYYY-MM-DD-开发总结.md`

**示例**：
- `2025-01-06-开发总结.md`
- `2025-01-07-开发总结.md`

### Checkpoint记录/
存放各个开发阶段的 Checkpoint 记录，记录重要里程碑和检查点。

**文件命名格式**：`Checkpoint-序号-描述.md` 或 `YYYY-MM-DD-Checkpoint-描述.md`

**示例**：
- `Checkpoint-01-项目初始化完成.md`
- `2025-01-06-Checkpoint-首页开发完成.md`

## 使用建议

### 每日总结模板

```markdown

## 今日完成
- [x] 完成项目初始化
- [x] 配置 Next.js 和 Contentlayer

## 今日进展

- 基础配置已完成
- 开始开发首页组件

## 遇到的问题
```

```markdown

**阶段**: 项目初始化
```

## 记录规范

### 每日总结记录规范

1. **每日记录**：每天开发结束后记录当日总结
2. **格式统一**：使用统一的模板格式
3. **内容完整**：包含完成、进展、问题、计划
4. **及时更新**：及时记录，避免遗忘

### Checkpoint 记录规范

1. **重要节点**：在重要开发节点记录 Checkpoint
2. **验证清单**：包含验证清单，确保质量
3. **可追溯**：记录相关文件和文档
4. **清晰描述**：清晰描述完成内容和下一步计划
