# 修复 Vercel 404 错误

## 🔍 问题原因

404 错误是因为 Vercel 从仓库根目录开始构建，但项目代码在 `01-源代码/网站代码/` 子目录中。

## ✅ 解决方案（两种方法）

### 方法 1: 在 Vercel 项目设置中配置（推荐，最简单）

1. **访问 Vercel 项目**
   - 打开 https://vercel.com
   - 选择你的项目 `sandy-ai`

2. **进入设置**
   - 点击顶部导航栏的 **"Settings"**
   - 在左侧菜单点击 **"General"**

3. **配置 Root Directory**
   - 找到 **"Root Directory"** 部分
   - 点击 **"Edit"** 按钮
   - 输入：`01-源代码/网站代码`
   - 点击 **"Save"**

4. **重新部署**
   - 配置保存后会自动触发新部署
   - 或手动点击 "Deployments" > "Redeploy"

### 方法 2: 使用根目录 vercel.json（已创建）

我已经在仓库根目录创建了 `vercel.json`，它会：
- 自动切换到 `01-源代码/网站代码/` 目录
- 执行安装和构建命令

**下一步**：推送这个文件到 GitHub

```bash
cd "/Users/luyu/CascadeProjects/Sandy的AI收藏夹"
git push
```

推送后，Vercel 会自动重新部署。

## 🎯 推荐操作

**推荐使用方法 1**（在 Vercel 设置中配置），因为：
- ✅ 更简单直接
- ✅ 不需要修改代码
- ✅ 配置更清晰

**如果方法 1 不行，使用方法 2**（已创建的 vercel.json）

## 📝 验证修复

修复后，访问网站应该：
- ✅ 不再显示 404 错误
- ✅ 能看到首页内容
- ✅ 所有页面正常访问

## 🆘 如果仍然有问题

1. 检查 Vercel 构建日志，查看是否有错误
2. 确认 Root Directory 路径正确
3. 确认项目结构正确
