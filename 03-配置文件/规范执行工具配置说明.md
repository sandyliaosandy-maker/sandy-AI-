# 规范执行工具配置说明

## 配置文件位置

所有配置文件应放在项目根目录（`01-源代码/网站代码/`），但可以在这里作为参考和备份。

## 配置步骤

### 步骤 1: 安装依赖

```bash
cd 01-源代码/网站代码
npm install --save-dev \
  eslint \
  eslint-config-next \
  @typescript-eslint/parser \
  @typescript-eslint/eslint-plugin \
  prettier \
  husky \
  @commitlint/cli \
  @commitlint/config-conventional
```

### 步骤 2: 创建配置文件

按照 `规范执行机制.md` 中的配置示例创建配置文件。

### 步骤 3: 配置 Git Hooks

```bash
npx husky install
npx husky add .husky/pre-commit "npm run check"
npx husky add .husky/commit-msg 'npx --no -- commitlint --edit ${1}'
```

### 步骤 4: 配置 VS Code

创建 `.vscode/settings.json` 和 `.vscode/extensions.json`。

### 步骤 5: 测试配置

```bash
npm run check
```

## 注意事项

1. 配置文件应在项目初始化时创建
2. 配置文件应与代码规范文档保持一致
3. 配置文件应提交到 Git 仓库
4. 团队成员应使用相同的配置
