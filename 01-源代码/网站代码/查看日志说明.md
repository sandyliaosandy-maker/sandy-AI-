# 如何查看服务器终端日志

## 方法 1: 在前台运行服务器（推荐）

在终端中运行以下命令，这样可以看到所有实时日志：

```bash
cd "/Users/luyu/CascadeProjects/Sandy的AI收藏夹/01-源代码/网站代码"
npm run dev
```

这样服务器会在前台运行，所有日志（包括错误信息）都会直接显示在终端中。

按 `Ctrl + C` 可以停止服务器。

## 方法 2: 查看最近的错误日志

如果服务器在后台运行，可以查看 Next.js 的错误日志：

```bash
cd "/Users/luyu/CascadeProjects/Sandy的AI收藏夹/01-源代码/网站代码"
cat .next/trace 2>/dev/null || echo "没有找到 trace 文件"
```

## 方法 3: 使用日志重定向

启动服务器时将输出保存到文件：

```bash
cd "/Users/luyu/CascadeProjects/Sandy的AI收藏夹/01-源代码/网站代码"
npm run dev 2>&1 | tee server.log
```

然后可以在另一个终端查看日志：

```bash
tail -f "/Users/luyu/CascadeProjects/Sandy的AI收藏夹/01-源代码/网站代码/server.log"
```

## 方法 4: 检查浏览器控制台

打开浏览器开发者工具（F12），查看 Console 和 Network 标签页，可以看到客户端错误和网络请求错误。

## 当前问题排查

如果看到 500 错误，通常会在服务器日志中显示：
- Contentlayer 生成错误
- 模块导入错误
- 类型错误
- 运行时错误

**建议**：使用方法 1 在前台运行服务器，这样可以直接看到所有错误信息。
