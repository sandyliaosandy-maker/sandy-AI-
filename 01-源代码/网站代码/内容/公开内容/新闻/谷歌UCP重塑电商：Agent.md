# 谷歌UCP重塑电商：Agent

**来源**: InfoQ中文

## 金句

💎 品牌不再争夺用户注意力，他们将竞相争取被Agent选中。
💎 网站变得可有可无，这就是非人类商业的开端。
💎 内容爆炸与上下文，是AI时代商品可发现性的阴阳两极。
💎 页面不再是交易的唯一入口，商品数据本身开始成为入口。

## 水下信息

• 谷歌UCP协议的核心野心并非仅仅是标准化购物流程，其真正目标是成为AI时代商业交易的底层基础设施（类似HTTP之于Web），这将从根本上改变电商的流量入口和竞争规则，使“网站”本身的重要性下降，品牌竞争将转向“被AI Agent选中”。
• UCP协议中尚未正式发布的“购物车”模块，被行业内部人士解读为意在挑战电商领域的“圣杯”——实现跨商家、多商品、由商家作为交易主体的统一购物车，即“一个购物车管全网”。这暗示了谷歌可能正在构建一个超越现有平台壁垒的终极购物体验。
• 在AI/Agent时代，商品信息的优化逻辑发生根本性转变。过去针对人类SEO（关键词、要点）的优化策略将失效，AI需要的是“内容爆炸+上下文”，即极其丰富、结构化的产品属性数据和关联信息。商家需要疯狂扩充商品级内容（如新增数十个字段，涵盖规格、兼容性、配件等），这直接决定了在AI搜索中的排名和“被选中”的概率。
• 谷歌正在利用其与零售商的现有深厚关系（如AdWords、广告投放）作为推广UCP的杠杆，这是一种典型的“生态绑定”策略。这意味着早期接入的合作伙伴可能在未来的AI商业流量分配中占据先发优势，形成新的竞争壁垒。
• 行业内部观察指出，像阿里巴巴这样的巨头虽然其关联公司（蚂蚁金服）已接入UCP，但很可能同时也在秘密研发自己的AI Agent商业协议和平台，以防在未来的标准之争中受制于人。这揭示了头部玩家在拥抱开放标准的同时，仍在进行“防御性”或“替代性”的底层技术布局。

## 案例提取

1. 全球范围 谷歌通过发布并开源Universal Commerce Protocol（UCP）标准，为零售商、电商平台及支付服务商提供服务。该协议旨在为AI智能体（Agent）提供一套标准化的端到端购物流程构建模块，以实现跨平台的互操作购物。协议采用Apache 2.0许可证在GitHub开源，发布首日即吸引了超过20家合作伙伴加入。风险投资人Linas Beliūnas评论称，UCP试图成为商业领域的“HTTP”，是AI介导交易不可或缺的基础设施。

2. 美国 谷歌通过升级“Buy for Me”结账功能，为使用其AI产品的消费者提供服务。新结账入口将出现在搜索AI Mode和Gemini应用中，将购物流程压缩为“商品 → 确认订单 → 下单完成”三步。该功能将率先在美国上线。长期关注零售的创业者Scott Wingo认为，这是对电商“圣杯”——跨商家统一购物车的一次强信号挑战。

3. 全球范围 谷歌通过扩展其Merchant Center的商品数据规范，为使用其服务的品牌和零售商提供服务。新增了数十个描述性文本、产品规格、兼容性信息等字段，以提供AI时代所需的内容与上下文，提升商品在AI搜索中的可发现性。Scott Wingo估计新增字段数量在24-60个之间。他建议品牌尽可能扩展商品级内容，这将直接决定其能否在AI时代被Agent选中。

4. 中国 阿里巴巴通过在其千问App中上线“任务助理”AI Agent能力，为淘宝、闪购、飞猪等应用的用户提供服务。该Agent能根据用户指令（如“我要两杯奶茶”）自动完成选店、选地址、选商品并生成订单，用户最后确认支付即可。该功能于2024年1月15日上线。内部测试导致阿里园区被大量奶茶订单包围，显示了其实际应用效果。
