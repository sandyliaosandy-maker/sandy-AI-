# MiniMax发布AI工作台A

**来源**: 量子位

## 金句

💎 理想的Agent，应该像个老练的同事，用得越久就越懂你。
💎 曾经那些将需求、设计与代码强行割裂的流水线，正在快速消融。

## 水下信息

• MiniMax内部已将Agent作为“实习生”深度融入日常工作流，例如Global Business组用其自动生成针对20个不同国家客户的定制化破冰邮件，设计组通过“Vibe Coding”直接指挥Agent修改前端代码，产品经理让其自动调查后台日志并拟定用户补偿方案，研发部门让其自动拆解任务、编写基础代码和测试。这揭示了AI Agent在顶尖公司内部已从概念验证进入大规模实战应用，并形成了具体的高效协作模式。
• Agent 2.0的Desktop App在执行复杂任务时，采用了“串并联混合”的搜索策略来处理批量查询（如同时调查20款产品），而非简单的顺序或全并发。这种策略平衡了速度与系统负载，是优化AI Agent执行效率、避免被反爬或请求限制的实用工程技巧。
• 要让AI Agent真正成为“长期合伙人”，关键在于其必须具备对“隐性经验”的消化能力，即能将行业内部标准、团队独有操作流程等非公开文档化的知识内化为行动本能。这暗示了成功部署企业级Agent的核心障碍并非技术，而在于如何系统性地将这些隐性知识抽取和结构化。
• Expert Agents功能允许用户上传私有知识库（如资深SEO玩法、量化交易策略）来创建垂直领域专家，这实际上提供了一种将个人或团队的核心竞争力“封装”成AI工作能力的路径。这是一种将隐性知识资产化、并防止其因人员流动而流失的潜在策略。
• 文章通过一个详细的测试案例（假装投资人调研20款Micro-SaaS产品）揭示了Desktop App的完整工作链条：读取本地文件 -> 转换任务清单 -> 智能分组联网检索 -> 结构化整理数据 -> 回写本地文件 -> 生成Markdown报告 -> 调用专家模块合成PPT。这提供了一个近乎标准化的、可复制的复杂信息处理与报告生成自动化流程范本。

## 案例提取

1. 中国 MiniMax公司通过其AI原生工作台Agent 2.0的Desktop App功能，为关注Micro-SaaS领域的投资人（模拟）提供服务。Desktop App自动联网检索20款产品的商业模式、流量来源等信息，将数据填回本地表格并生成图文报告和PPT。成功完成对20款产品的深度调研和信息填充，生成内容经第三方验证全部正确。用户（模拟）通过此过程验证了Agent直接调取本地资源并完成复杂、多环节任务的能力。

2. 中国 MiniMax公司通过其AI原生工作台Agent 2.0，为全球业务组员工Morgan提供服务。Agent根据一份涉及20个不同国家潜在客户的名单，自动进行背景调查并生成定制化的破冰邮件。瞬间生成了20封完全定制化的邮件，将原本需要耗时一天的背景调查工作高效完成。Morgan得以快速处理大量客户破冰工作。

3. 中国 MiniMax公司通过其AI原生工作台Agent 2.0，为设计组员工Joanna提供服务。Joanna使用“Vibe Coding”方式，在编辑器中直接指挥Agent修改官网字体等前端代码。Agent自动滚动修改代码，网页样式瞬间完成变更。Joanna作为设计师无需等待前端排期，即可快速实现设计调整。

4. 中国 MiniMax公司通过其AI原生工作台Agent 2.0，为产品经理寻鹭提供服务。Agent介入调查用户关于“积分消耗不合理”的投诉，自动排查后台系统并查明原因（用户上传37个超大文件），随后根据公司政策拟定安抚方案并起草邮件。查明原因是用户上传了37个超大文件导致Token消耗飙升，并主动拟定“退还20%积分”的方案。寻鹭全程仅需回复确认，高效解决了用户纠纷。

5. 中国 MiniMax公司通过其AI原生工作台Agent 2.0，为研发部门（如计算平台组）的工程师提供服务。Agent通读运维手册，在系统报警时自动分析原因并给出处理建议；或根据需求列表自动拆解任务、编写与测试基础代码。Agent能够接管绝大部分基础代码的编写与测试工作。研发人员的角色得以重塑，工作效率提升。
