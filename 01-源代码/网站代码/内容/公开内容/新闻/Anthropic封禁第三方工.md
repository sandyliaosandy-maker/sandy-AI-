# Anthropic封禁第三方工

**来源**: InfoQ中文

## 金句

💎 模型能力不再是唯一焦点，控制使用方式才是博弈核心。
💎 Anthropic提供的是“无限自助餐”，但前提是你吃得慢。
💎 订阅模式本身并不盈利，而更像是一个典型的“亏损引流产品”。
💎 长期的成功不仅取决于最佳模式，还取决于你如何与开发者互动。
💎 要求将核心工具完全开源，无异于让公司主动削弱自身竞争壁垒。

## 水下信息

• Anthropic的服务条款D.4节明确禁止使用其服务来“构建或训练竞争性AI系统”，以及“对服务进行逆向工程或复制”。xAI工程师通过Cursor使用Claude加速自家模型研发，被认定为“竞争性使用”，这是其访问权限被切断的直接法律依据，揭示了AI公司间利用服务条款进行商业防御的潜规则。
• 第三方工具（如OpenCode）通过模拟官方客户端（Claude Code CLI）的身份、伪造请求头和复用OAuth Token来实现“客户端欺骗”，从而绕过官方速率限制和计费机制，将按对话设计的订阅制模型转化为可被自动化代理低成本、高强度调用的基础设施。这是技术社区中一种游走在灰色地带的通用实现方案。
• Claude Pro/Max（最高200美元/月）的订阅定价是基于“人类交互速率”设计的，其成本与按token计费的API存在巨大断层。在高频编程场景下，通过订阅消耗的token若全部走API，成本可能轻松超过1000美元。因此，该订阅实质上是一个“特殊定价的官方通道”或“亏损引流产品”，其真实商业意图是引导用户使用官方生态并实现厂商锁定，而非提供真正的“无限量自助餐”。
• AI模型提供商（如Anthropic）对通过第三方工具调用模型的行为保持警惕，核心原因之一是技术不可控与责任归属问题：当封装器内部出现错误或性能问题时，用户会归咎于“模型变差”，而提供商无法有效复现和诊断，这损害了模型声誉和用户体验的可控性。
• 在AI编程工具市场，各家公司竞争的核心已不仅是模型能力，更是对模型使用方式、定价结构和开发者通道的控制权。Anthropic此次行动，连同此前切断Windsurf、OpenAI的访问，表明其战略是保护其核心利润市场（编程），并通过控制“桥梁”（集成路径）来确保用户流向其官方工具和生态系统，而非单纯销售模型能力。

## 案例提取

1. 全球 Anthropic通过提供Claude Pro/Max订阅服务为开发者提供服务。该服务允许用户以固定月费（最高200美元/月）使用Claude模型进行编程辅助，其定价基于“人类交互速率”设计，并包含速率限制。每月订阅费用为100至200美元。部分开发者认为该订阅模式与API计费相比存在成本断层，在高频使用下通过API可能需花费超过1000美元。

2. 全球 OpenCode团队通过开发开源AI编码助手为开发者提供服务。该工具将Claude模型集成到VS Code、Cursor等IDE中，增加了键盘快捷键、上下文感知和多文件编辑功能，旨在将AI推理融入开发者现有工作流程。工具版本为1.1.8及以上。开发者非常喜欢该工具，认为其提供了更好的工作流程集成和可见性，称仅使用官方Claude Code如同“回到石器时代”。

3. 全球 Cursor通过集成多种AI模型为开发者（包括xAI的内部开发人员）提供服务。该IDE允许开发者调用包括Claude在内的AI模型来加速编码、研发与测试工作。服务对象包括竞争对手xAI的工程师。xAI工程师曾使用该工具调用Claude加速自家模型研发，后因违反Anthropic服务条款的竞争限制而被切断访问。

4. 全球 Windsurf编码环境在2025年6月后通过转向BYOK（自带密钥）并主推Gemini模型为用户提供服务。在突然被Anthropic切断对Claude 3.x模型的直接访问后，该环境调整了服务模式，为用户提供替代的AI编程助手方案。关键时间点为2025年6月。此举是迫于Anthropic政策变化的应对措施，文中未提供具体的用户反馈数据。

5. 全球 Anthropic在2025年8月通过执行服务条款撤销了OpenAI对Claude API的访问权限。Anthropic以OpenAI将Claude用于模型基准测试与安全评估（构成竞争性使用）为由，采取了此次执法行动。关键时间点为2025年8月。Anthropic对此解释直白，认为竞争对手使用其工具“并不令人意外”，但“使用”和“竞争性使用”有清晰界线。
