# AI编程进化：从代码补全到全托

**来源**: 少数派

## 金句

💎 淘汰你的从来不是熊，而是比你跑得更快的人。
💎 优秀的工程师不会消失，只是工作职责变了。
💎 我们应当保持对AI的信任，但必须坚守信任但要核实的原则。
💎 熟练掌握AI Coding Agent已不再是加分项，而是生存技能。

## 水下信息

• 对于本地AI Coding Agent（如Claude Code、Codex）的“全托管”模式，存在需要手动开启的、具有潜在破坏性的高级权限开关，例如 `--dangerously-skip-permissions` 和 `--dangerously-bypass-approvals-and-sandbox`，这通常是官方文档中不会重点强调的实践操作。
• 作者透露了不同身份背景的人使用AI Coding Agent的真实困境：对于没有软件工程基础的小白用户，他们经常卡在基础问题上，而解决这些问题的关键不是继续研究AI工具，而是去学习《计算机教育中缺失的一课》这类基础课程，这是从实践中得出的、针对特定人群的有效学习路径。
• 在深度使用AI Coding Agent时，必须建立一套“质量守卫”体系来对冲AI的不可靠性，具体包括：利用git hook强制代码格式化、要求100%测试覆盖率（包含集成和端到端测试）、集成linter工具让AI自愈问题、引入圈复杂度检查工具控制代码质量。这套组合拳是保障项目可控性的核心内幕。
• 作者分享了个人付费使用策略：即使公司不报销，也强烈建议个人购买Claude Code等服务，并明确指出Claude Code 5X版本已能满足日常使用，用量大再升级到20X版本，将这笔开销（约100美元）类比为一顿饭钱，强调其投资回报率远超价格，这是一种基于实际体验的成本效益分析。
• 通过同时使用多个不同的Coding Agent（如Claude Code、OpenAI Codex、Gemini Cli），可以利用不同模型的视角差异来弥补单一模型的思维漏洞，这是一种提升代码质量和自我思考维度的实战技巧，而非单纯的功能叠加。

## 案例提取

1. 个人工作环境 作者通过使用Claude Code的Plan Mode进行本地开发，为自己（一名产品经理兼工程师）提供服务。作者让AI设计需求、约束范围并完成相关功能开发。在Claude Sonnet 3.7版本后，作者开始将更多工作交给AI并只做关键验收。用户反馈是AI Coding Agent的使用频率与日俱增，并最终对AI放权，开启了全托管模式。

2. 个人工作环境 作者通过使用OpenAI Codex为项目补全测试，为自己（一名工程师）提供服务。利用AI帮助完成项目测试从0到1的建设，覆盖基础测试。过去作者常懒得写测试，现在效率大大提升。用户反馈是AI虽无法像专业测试人员那样完善，但能有效帮助建立测试围栏。

3. 个人工作环境 作者通过在CI工具链中集成多个AI Coding Agent（如Claude Code、Codex），为自己（一名工程师）的代码提交和PR提供代码评审服务。集成多个Agent以获得不同视角，发现代码问题和思考漏洞。用户反馈是这能确保思考无漏洞，补全思考维度，并对自我提升非常有帮助。

4. 个人工作环境 作者通过引入包括静态检查、质量红线、AI自愈能力和代码复杂度分析在内的质量守卫机制，为自身深度使用AI Coding Agent的项目提供服务。确保AI提交的代码可信可控，无低级错误和逻辑错误。在强AI介入的项目中要求测试覆盖率达到100%。用户反馈是有了充足的围栏检查后，可以放心让AI完成工作并自动提交。
