# Claude Code创始人揭

**来源**: InfoQ中文

## 金句

💎 开发者从敲代码语法转变为指挥自主协作的智能单元。
💎 每一次失误都能转化为一条规则。
💎 智能体不只是编写代码，更能主动验证代码的有效性。
💎 将重复性任务进行了极致的自动化改造。

## 水下信息

• 在终端并行运行多个AI智能体（如5个Claude实例）并利用系统通知（如iTerm2通知）来管理其状态和输入指令，可以模拟“舰队指挥官”式的工作模式，将编程从线性任务转变为并行战略游戏，从而大幅提升单人产出效率。
• 在追求低延迟的AI编程领域，反直觉地坚持使用最大、最慢的模型（如Claude Opus 4.5）并开启其“思考模式”，可能因为其更强的工具调用能力和更少的人工干预需求，反而在整体效率上超越更小、更快的模型（如Sonnet）。
• 通过维护一个名为`CLAUDE.md`的共享文件来记录AI在特定代码库中犯过的错误和学到的规则，可以系统性地解决大语言模型的“健忘症”问题，让AI在后续任务中避免重复犯错，使代码库成为一个“能够自我修正的有机体”。
• 为AI智能体赋予自主验证工作成果的能力（如通过浏览器扩展自动运行测试、验证UI），而不仅仅是生成代码，可以将最终产出质量提升2-3倍。这包括自动化测试套件、浏览器操作和Shell命令执行。
• 通过创建并集成自定义斜杠指令（如`/commit-push-pr`）到项目代码库中，可以将Git提交、推送、创建拉取请求等繁琐的版本控制流程一键自动化，每天高频次使用以极致优化重复性任务。

## 案例提取

1. 线上/远程 Boris Cherny通过并行运行多个Claude智能体并利用系统通知进行管理，为自身软件开发工作提供服务。他同时指挥五个智能体分别执行运行测试、重构模块、撰写文档等不同任务，将编程转变为战略指挥。他每天调用自定义指令/commit-push-pr数十次以自动化版本控制流程。用户反馈其工作流体验“更像玩《星际争霸》”，并能发挥出小型工程团队的产出效能。

2. 线上/远程 Boris Cherny及其团队通过维护名为CLAUDE.md的共享文件，为团队及AI智能体提供服务。该文件用于记录Claude执行任务时出现的错误和解决方案，使AI能学习并避免重复犯错，将代码库变为可自我修正的有机体。关键做法是每当人类开发者审核代码发现错误时，不仅修正代码还会标记问题以更新AI指令。分析人士评价“每一次失误都能转化为一条规则”。

3. 线上/远程 Boris Cherny通过为AI智能体赋予自主验证能力，为其代码修改工作提供服务。他利用Claude浏览器扩展，使AI在提交每一处修改后能自动打开浏览器测试用户界面并迭代优化，直至代码正常运行且用户体验达标。他认为此举能将最终产出质量提升2~3倍。核心服务内容是让智能体不仅编写代码，更能主动验证代码的有效性。

4. 线上/远程 Boris Cherny通过部署专用子智能体，为其软件开发生命周期中的特定阶段提供服务。在核心开发完成后，他会启用代码简化智能体来优化架构；在正式发布前，则调用应用验证智能体执行端到端测试。此举是其极致自动化改造重复性任务工作流的一部分。观察人士盛赞这套“原生”工作流的核心优势在于对重复性任务的自动化改造。
